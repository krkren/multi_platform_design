Layer.hasImage

Function/Meaning
Whether the layer has an image
Type
Property of the Layer class
Description

Indicates whether the layer has an image.
This value can also be set.
If set to true, the layer will have an image.
This is the default.
If set to false, the layer's image is released and the layer will no longer have an image.
Setting the Layer.type property resets hasImage to true.
If the layer does not have an image and Layer.type is ltOpaque, the layer is displayed as being completely filled with Layer.neutralColor.
For other types, it is treated as completely transparent.
If this property is false and Layer.hitType is htMask, the entire layer is considered to have an opacity of 0 (completely transparent).
Additionally, drawing and font operations cannot be performed.
Layers with this property set to false are typically used to group multiple child layers while remaining transparent themselves.
