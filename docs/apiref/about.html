マルチプラットフォーム版 について

Windowsに加えてAndroidでも動作するようにしたバージョンです。
非力な環境でも描画速度が出るようにGPU描画がメインとなっています。
現在のところ旧バージョンとは互換性が一部ありません。

About the Multi-Platform Version

This is a version that runs on Android in addition to Windows.
GPU rendering is the main focus to achieve drawing speed even in weak environments.
Currently, there is some incompatibility with the old version.

環境依存機能

マルチプラットフォーム版は全環境で全てのメソッドやプロパティが使えるわけではありません。
一部機能については環境固有となります。
環境固有のメソッド等については、説明概要に[Windows]や[Android]と文頭に書かれています。
そのように書かれているメソッド等はその環境でのみ使用できます。
呼び出しても何も起こらないもの、例外が出るものなどあります。
詳細については、以下のように末尾記号で判別してください。

Environment-Dependent Functions

Not all methods and properties can be used in all environments in the multi-platform version.
Some functions are environment-specific.
For environment-specific methods, etc., [Windows] or [Android] is written at the beginning of the description outline.
Methods etc. written in that way can only be used in that environment.
Some do nothing when called, and some throw exceptions.
For details, please use the following suffix symbols to determine.

  • [Windows+] Windowsでのみ使用可能、その他環境では例外や動作不定などがあるので使用禁止。
  • [Windows*] Windowsで使用可能、その他環境では何も起こらない。
  • [Windows] Windowsで使用可能、その他環境では動作不定。
  • [Android+] Androidで使用可能、その他環境では例外や動作不定などがあるので使用禁止。
  • [Android*] Androidで使用可能、その他環境では何も起こらない。
  • [Android] Androidで使用可能、その他環境では動作不定。
  • [Windows+] Usable only on Windows, prohibited in other environments because exceptions or undefined behavior may occur.
  • [Windows*] Usable on Windows, does nothing in other environments.
  • [Windows] Usable on Windows, behavior is undefined in other environments.
  • [Android+] Usable only on Android, prohibited in other environments because exceptions or undefined behavior may occur.
  • [Android*] Usable on Android, does nothing in other environments.
  • [Android] Usable on Android, behavior is undefined in other environments.

System.isWindowsやSystem.isAndroidプロパティを用いることで動的にどちらの環境か判定することができます。
静的に判定する場合は、プリプロセッサを用います。
Windowsでは"windows"が1に、Androidでは"android"が1に設定されています。
静的に判定する場合、バイトコードの出力結果が環境に依存することに注意してください。
コンパイルした環境用のバイトコードが出力されます。
これを回避するためにはコンパイル関数呼び出しの前にプリプロセッサの値を事前に変更し、各環境に合ったものにしてバイトコード化する必要があります。

You can dynamically determine which environment you are in by using the System.isWindows and System.isAndroid properties.
To determine statically, use the preprocessor.
In Windows, "windows" is set to 1, and in Android, "android" is set to 1.
When determining statically, note that the output result of the bytecode depends on the environment.
Bytecode for the compiled environment will be output.
To avoid this, it is necessary to change the value of the preprocessor in advance before calling the compile function, make it suitable for each environment, and convert it into bytecode.
