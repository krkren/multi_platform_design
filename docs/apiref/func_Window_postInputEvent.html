Window.postInputEvent

Function/Meaning

Generation of input events

Type
Method of the Window class
Syntax
postInputEvent(eventname, params=null)
Arguments
eventname

Specifies the event name.
Specify using the following strings.

  • "onKeyDown" generates a Window.onKeyDown event.
  • "onKeyPress" generates a Window.onKeyPress event.
  • "onKeyUp" generates a Window.onKeyUp event.

Since onKeyDown and onKeyUp are pairs, it is recommended to generate the corresponding onKeyUp after generating an onKeyDown.

params

Specifies a dictionary array containing the event parameters.
For "onKeyDown" and "onKeyUp" events, store the virtual key code in "key" and the shift state in "shift".
If "shift" is omitted, it is assumed to be 0.
For "onKeyPress" events, specify the character in "key".

Return Value
None (void)
Description

Generates an input event.
In the current version, three events related to key input can be generated.
This method generates the event as an asynchronous event.
In other words, this method returns without waiting for the corresponding event handler to finish.
The actual event handler call and processing occur after control returns to Kirikiri.
Like normal input events, input events occur not only for the Window class but also as the corresponding events for the Layer class.
Example:


postInputEvent('onKeyDown', %[key: VK_UP, shift: ssShift]);
postInputEvent('onKeyUp', %[key: VK_UP, shift: ssShift]);
// Press the up cursor key
