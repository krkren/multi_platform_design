WaveSoundBuffer_PhaseVocoder.window

Function/Meaning
Window size
Type
Property of the WaveSoundBuffer_PhaseVocoder class
Description

Represents the window size (frame size) of the FFT (Fast Fourier Transform) used internally by the PhaseVocoder.\n Please specify a power of 2 between 64 and 32768. The default is 4096. Increasing this value improves resolution in the frequency domain (making pitch more accurate) but decreases resolution in the time domain (making the attack of the sound duller). Larger values will use more CPU. Please find the lowest value that maintains sound quality. For 44.1KHz voice, a window size of about 256 is recommended. For music, a window size of about 4096 is recommended. Changes to this property are reflected when the WaveSoundBuffer.open method is executed.
