Layer.independMainImage

Function/Meaning

Unsharing the main image

Type
Layer class method
Syntax
independMainImage(copy=true)
Arguments
copy

Specifies whether to copy the original image when unsharing.
If true, the original image is copied. If false, the original image is not copied and the image content becomes undefined.

Return Value
None (void)
Description

Forcibly unshares the layer image.
In Kirikiri, when an entire image is copied to another layer using assignImages or similar methods, the image buffer is not actually copied; instead, the same image is shared.
Normally, this sharing state is automatically released just before an attempt is made to modify the image, but it can be forcibly released with this method.
If false is specified for the copy argument, the image sharing is released, but there is no guarantee that the original image will be preserved (the image content becomes undefined), though the sharing can be released more quickly.
If you are going to overwrite the entire layer image, there is no need to preserve the original image, so calling this method with false before drawing may improve efficiency.
This method does nothing if the image is not shared.
