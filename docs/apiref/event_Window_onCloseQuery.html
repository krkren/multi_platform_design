Window.onCloseQuery

Function/Meaning

[Windows*] Confirmation for closing the window

Type
Window class event
Syntax
onCloseQuery(canclose)
Arguments
canclose

Passes whether the window can be closed.
In the process of calling event handlers from a subclass to a superclass, the decision made by the subclass regarding "whether the window can be closed" is passed.

Return Value
None (void)
Description

This event is for confirming whether the window can be closed.
If the window cannot be closed, pass false as an argument to the same method of the superclass.
On Android, you must handle termination yourself (by handling onKeyXXX) when the back key is pressed on the title screen.
When the system forces a stop, it cannot be prevented and the application may be terminated abruptly.
Termination from the task list is also immediate and cannot be stopped.
On Android, the only countermeasure against sudden termination is to perform an auto-save when onDeactivate is received.
