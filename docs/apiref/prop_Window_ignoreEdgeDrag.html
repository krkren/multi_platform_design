Window.ignoreEdgeDrag

Function/Meaning
Whether to ignore input when touched from the surrounding edges. Defaults to true on Android. (Unimplemented)
Type
Property of the Window class
Description

On Android, if coming from the left, VK_MENU KeyDown/Up is sent. Should we use that as a trigger to show the menu? Do we want it to slide out gradually?
Add something like MenuDragging to the shift state of onPointerDown, use that for detection, and show the menu from the left?
On Android, the size is defined according to the screen resolution (density) as follows:
int EDGE_SIZE = 20;
mEdgeSize = (int) (EDGE_SIZE * density + 0.5f);
