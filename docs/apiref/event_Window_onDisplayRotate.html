Window.onDisplayRotate

Function/Meaning

When the screen is rotated

Type
Event of the Window class
Syntax
onDisplayRotate(orientation, angle, bpp, width, height)
Arguments
orientation

The screen orientation. It will be one of the following values: oriUnknown (failed to get/unknown), oriPortrait (portrait), oriLandscape (landscape)

angle

The angle. The angle is one of 0, 90, 180, 270, or -1. It is -1 if it could not be obtained.
The angle is the rotation angle from the device's default, so for a portrait device, portrait is 0.
For typical devices, landscape is often 0.
Portrait becomes 0 on recent 8-inch tablets where portrait is the standard orientation.
On Android, -1 is always returned.
If necessary, obtain it separately via Window.displayRotate.

bpp

Bits per pixel. On Android, the dpi value is returned.

width

The screen width.
On Android, 0 is always returned.
Obtain it separately if necessary.

height

The screen height.
On Android, 0 is always returned.
Obtain it separately if necessary.

Return Value
None (void)
Description

Represents an event function called when the screen is rotated.

Reference
Window.displayOrientation
