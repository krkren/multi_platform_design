Layer.loadImages

Function/Meaning

Loading images

Type
Layer class method
Syntax
loadImages(image, colorkey=clNone)
Arguments
image

Specifies the image storage to load.
If an image storage exists with the specified storage name (excluding extension) plus _m, it will be loaded as a mask (opacity) image.
If an image storage exists with the specified storage name (excluding extension) plus _p, it will be loaded as a province (region) image.

colorkey

Specifies the color key (transparent color) of the image to be loaded.
If a color is specified in 0xRRGGBB format, that color is used as the color key. If a value consisting of clPalIdx plus an arbitrary palette index is specified, that palette index becomes the transparent color (for images with 256 colors or less).
If clAdapt is specified, the most frequently used color in the top line of the image automatically becomes the transparent color.
If a value consisting of clAlphaMat plus a color in 0xRRGGBB format is specified, the image is overlaid on that color using alpha blending (ltAlpha method).
For example, if (clAlphaMat + 0xffffff) is specified, the loaded image will be overlaid on a white color.
All images will become opaque (note that while all images become opaque, tag information is not changed at all in this mode).

Return Value

Dictionary array of tag information

Description

Loads an image into the layer.
This method changes the image size of the layer, but does not change the display size of the layer unless the image size is smaller than the layer's display size.
A dictionary array of tag information (information about the image itself, such as the image's layer type and display position) is returned as the return value.
Be careful not to confuse this with the meaning of "tags" in KAG.
If the image does not have tag information, null is returned. In the current version, only PNG and TLG5/6 formats can have tag information.
For information that can be obtained, please refer to the Image Format Converter.
