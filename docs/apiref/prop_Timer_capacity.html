Timer.capacity

Function/Meaning
Maximum event capacity
Type
Timer class property
Description

Specifies how many events can be generated at one time.
You can also set this value.
When processing takes longer than the timer interval, timer events may not be processed in time.
In such cases, the timer stores events in the event queue, but there is a limit to the number of events that can be stored.
This property specifies that maximum number.
When it is time to process events, Kirikiri will attempt to process all events stored in the queue at once.
If set to 1, the event queue will store at most one event.
If set to 0, there is no limit to the number of events in the queue, but be careful as the system may appear to freeze if too many events accumulate.
The default is 6.
