Layer.getLayerAt

Function/Meaning

Get layer at specified position

Type
Layer class method
Syntax
getLayerAt(x, y, exclude_self=false, get_disabled=false)
Arguments
x

Specifies the x-coordinate of the layer position to be retrieved in pixels on the display coordinates.
The display coordinates of the layer executing this method are used (not the display coordinates on the primary layer).

y

Specifies the y-coordinate of the layer position to be retrieved in pixels on the display coordinates.
The display coordinates of the layer executing this method are used (not the display coordinates on the primary layer).

exclude_self

Specifies whether to exclude the layer itself from the search.
If false is specified, the layer itself is included in the search.
If true is specified, the layer itself is excluded from the search and treated as if it does not exist.
If this argument is omitted, false is assumed.

get_disabled

Specifies whether to obtain objects of disabled layers.
If false is specified, null is returned if a disabled layer (e.g., Layer.enabled property is false) is at the specified position.
If true is specified, the layer object is returned even if the layer at the specified position is disabled.
If this argument is omitted, false is assumed.

Return Value

The layer object at the specified position.
Returns null if no layer is at the specified position.

Description

Returns the layer object at the position indicated by x and y.
The hit test mechanism used is the same as for normal mouse events.
In other words, the layers are checked from the frontmost in the stacking order, and the first layer that satisfies the hit test is returned.
By specifying true for the exclude_self argument, you can exclude the layer executing this method from the search.
