System.touchImages

Function/Meaning

Loading images into cache

Type
Method of the System class
Syntax
touchImages(storages, limitbytes=0, timeout=0)
Arguments
storages

Pass an array (Array class object) of image storage names you want to put into the cache.
Items listed earlier have higher priority.
KiriKiri manages the cache using the string specified in the first argument of Layer.loadImages as the key. Therefore, for the cache to be effective, the image storage names specified here must be identical to those passed as the first argument to Layer.loadImages.

limitbytes

Specifies the limit for cache capacity used by this method call in bytes.
If 0 is specified, the entire cache is used. If a positive number is specified, it attempts to use the cache up to that number of bytes.
If a negative number is specified, the limit is calculated by adding that value to the current cache limit ( System.graphicCacheLimit ) (since it's a negative number, it effectively subtracts it).
If the resulting limit is 0 or negative, this method terminates without doing anything.
For example, if -2*1024*1024 is specified, it is treated as if the current cache limit minus 2MB was specified.
This is useful when you want to leave some free space in the cache.

timeout

Specifies the timeout (time limit) in ms. Specifying 0 makes it unlimited.
This method will stop loading subsequent images and return once the time specified by this argument has elapsed.
However, even if a timeout occurs while loading a specific image, it will not return until that image's loading is complete.

Return Value
None (void)
Description

This method attempts to load the specified images into the cache.
However, while this method makes an effort to put images into the cache, there is no guarantee that the images will actually be cached.
Image loading is interrupted if the image cache limit is exceeded or a timeout occurs.
Among the items specified in the storages argument, those listed earlier have a higher priority for being cached.
If a specified image is already in the cache, it simply raises its priority within the cache.
This method ignores all errors during image loading.
In the current version, images that can be cached with this method are those normally loadable via Layer.loadImages and which do not specify a color key (images with alpha channels are fine).
Rule images for universal transitions or region images will be loaded, but they will not become valid cache data (the loaded data will be wasted), so please do not specify them.
Refer to System.graphicCacheLimit for the number of bytes images use in the cache.
