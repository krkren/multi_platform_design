Layer.onHitTest

Function/Meaning

Hit test

Type
Layer class event
Syntax
onHitTest(x, y, hit)
Arguments
x

The x-coordinate of the position to perform the hit test (within the layer's display coordinates).

y

The y-coordinate of the position to perform the hit test (within the layer's display coordinates).

hit

The result of the hit test; true if hit, false if not.
Pass the result of the hit test to this argument of the superclass event of the same name.

Return Value
None (void)
Description

Called when a hit test is performed on a point within the layer.
Pass the result of the hit test to the third argument of this event in the superclass (pass the first and second arguments as they are).
If the hit test result is a hit, mouse messages become opaque and are processed by that layer.
If it is not a hit, it becomes transparent, and mouse messages are processed by layers further back.
This event is called after a "hit" determination is made by the hit test performed via Layer.hitThreshold or Layer.hitType.
Therefore, if Layer.hitThreshold or Layer.hitType determines it is "not a hit", this event will not be called.
