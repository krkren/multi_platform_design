Font.face

Function/Meaning
Font name
Type
Property of the Font class
Description

Represents the font name.
The value can also be set.
Multiple candidates can be specified by separating them with commas.
In this case, an existing font will be used, with priority given to those listed first.
Whether a font actually exists is determined by referring to the OS font list.
If none of the candidates match, the default font is used (the system standard font in the current version).
In the FreeType rasterizer, if multiple candidates are specified, they are treated as fallback fonts; glyphs are searched in order from the beginning, and the first glyph found is rendered.
This fallback font mechanism is used when font files exist as separate files for different languages or emojis.
If you start with a comma followed immediately by a font name, it will attempt to specify that font to the OS even if the OS does not enumerate it (for example, ",My Original Font" ).
This allows the use of non-enumerable fonts registered via the AddFontResource Win32 API or similar methods.
